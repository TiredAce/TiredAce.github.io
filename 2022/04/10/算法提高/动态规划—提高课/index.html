DP状态的表示：

题目需要从左上角开始走到右下角确定两条路径如何走，我们可以把走过的步数，作为DP的阶段。在每个阶段中，我们把两条路径同时拓展一步。设路径的长度为i。第一条路径末尾位于 ($x_1$,$y_1$)，第二条路径末尾位于 ($x_2$，$y_2$)。为了能找到能够覆盖整个状态空间的最小的 “维度集合”。得等式 $x_1 + y_1 = x_2 + y_2 = i + 2$

故 $i,x_1,x_2$三个维度即可描述为一个状态。

DP状态的转移：

dp[k][x1][x2]能从dp[k-1][x1][x2], dp[k-1][x1-1][x2], dp[k-1][x1][x2-1], dp[k-1][x1-1][x2-1]转移过来

272. 最长公共上升子序列

acwing原题链接

DP分析

状态表示：dp[i][j]表示所有的a[1~i]和b[1~j]中以b[j]结尾的公共上升子序列的集合。

状态计算：根据公共子序列中是否包含a[i]，将dp划分两类。

  • 不包含a[i]的子集，最大值时dp[i-1][j]。
  • 包含a[i]的子集，将这个子集继续划分，依据是子序列的倒数第二个元素在b[]中是哪个数。

python

复杂度算法：$O(n^3)$

C++

思路：可将第三层循环优化，我们发现每次循环求得的maxv是满足a[i] > b[k]的f[i-1][k] + 1的前缀最大值。因此可以直接将maxv提到第一层循环外，减少重复计算。

时间复杂度：$O(n^2)$

10. 有依赖的背包问题

DP分析

DP状态表示：dp[u][v]表示以u为根节点，背包体积为v时所获得的最大价值。

DP状态转移：根据依赖关系，每次都必须要将u选上，再考虑子树，结合分组背包的思想，将每种剩余的体积看成一组，然后看剩余体积分给各个子树，各个子树利用这些剩余体积能拼凑最大价值是多少。

734. 能量石

AcWing题库

算法分析：DP + 贪心

贪心部分

  • 对于最优方案的任意相邻的两个能量石i和j，贡献的能量总和为(假设t为之前的时间)Ei - t * Li + Ej - (t + Si) * Lj
  • 将i和j两个能量石交换，贡献的能量总数为Ej - t * Lj + Ei - (t + Sj) * Li
  • 交换后贡献总和不会变大，因此应满足Ei - t * Li + Ej - (t + Si) * Lj >= Ej - t * Lj + Ei - (t + Sj) * Li
  • 整理后得：Si * Lj <= Sj * Li将 i 的放在同一边即可得Si / Li <= Sj / Lj
  • 因此以Si / Li <= Sj / Lj排序后的能量石，就能保证最优解在子序列中

DP部分

  • dp数组的定义：dp[i][j]表示在前i个能量石中，耗时恰好为j得到的最大能量
  • dp数组的递推：情况一.不选当前的能量石能量为dp[j]; 情况二.选当前的能量石能量和为dp[j-s]+max(0,e-(j-s)*l)
